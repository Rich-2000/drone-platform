ℹ️ Upload Guidelines
01All images are stored permanently on Cloudinary CDN under the naimos/drones folder.
02Metadata (mission, region, GPS, drone ID) is stored as Cloudinary context tags and in local IndexedDB for offline access.
03The NAIMOS main platform at naimos.vercel.app fetches these images via the https://naimos.vercel.app/api/drones/images endpoint in real time.
04For in-flight uploads, configure the drone's ground station software to POST images to this platform's API endpoint automatically.
05Supported formats: JPG, PNG, TIFF, WebP, RAW (converted). Max 20MB per image. Batch uploads supported.
🔗 API Integration

Programmatic upload via the NAIMOS Drone API:

POST /api/drones/upload
Content-Type: multipart/form-data

Fields:
image: <file>
mission: "NAIMOS-M-2026-044"
region: "Western Region"
droneId: "NAIMOS-UAV-01"
lat: 5.8456
lon: -1.8234
type: "rgb"|"thermal"
notes: "..."

Response:
{ success: true, imageId, url, thumbnailUrl }

⚙️ API Configuration
ℹ The backend uses Cloudinary credentials from environment variables. The API base URL is configured here for the frontend client only.
📡 Backend Route Reference
GET https://naimos.vercel.app/api/drones/images — list all images + metadata
GET https://naimos.vercel.app/api/drones/images?region=Western+Region
GET https://naimos.vercel.app/api/drones/images?mission=NAIMOS-M-2026-044
GET https://naimos.vercel.app/api/drones/images?limit=20&after=TIMESTAMP
POST /api/drones/upload — multipart upload
POST /api/drones/upload-url — get signed upload URL
GET /api/drones/stats — fleet + image stats
DELETE https://naimos.vercel.app/api/drones/images/:id
